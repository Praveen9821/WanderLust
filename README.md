# 🌍 WanderLust – Full Stack Travel Listing Platform

[![Live Demo](https://shields.io)](https://wanderlust-yywd.onrender.com)]

WanderLust is a full-stack web application designed to help users explore, create, and manage travel destinations. It provides a seamless experience for discovering places, sharing reviews, and visualizing locations through interactive maps.

## 🚀 Project Highlights
* **MVC Architecture:** Built using a clean Model-View-Controller pattern for scalability.
* **Authentication & Authorization:** Secure user access and permission-based controls.
* **Geospatial Visualization:** Integrated Mapbox for real-time location mapping.
* **Database Management:** Scalable backend powered by MongoDB and Express.
* **Responsive UI:** Modern, mobile-friendly design using EJS and Bootstrap.

## 🧩 Core Features
### 🏠 Listings Management
* Create, edit, and delete travel listings with ease.
* Cloud-based image management for high-quality property photos.
* Detailed destination views with descriptions and pricing.

### ⭐ Reviews & Ratings
* Robust review system for community feedback.
* Star-rating integration for quick evaluation.
* Seamless linking between users, reviews, and listings.

### 🔐 Authentication & Security
* User signup, login, and logout functionality.
* Password hashing and session management using **Passport.js**.
* Custom middleware for route protection and authorization.

### 🗺 Map Integration
* Interactive maps using the **Mapbox API**.
* Automatic geolocation based on user input.
* Custom markers to visualize listing locations.

### ⚙️ Error Handling & Validation
* Strict server-side validation using **Joi**.
* Centralized error-handling middleware for a smooth user experience.
* Clean async error management with custom wrappers.

## 🏗️ Architecture
WanderLust follows the **MVC** pattern with a modular folder structure:
* `controllers/` – Business logic and request handling.
* `models/` – Database schemas and data structure.
* `routes/` – API endpoint definitions.
* `views/` – Dynamic UI templates (EJS).
* `public/` – Static assets (Stylesheets, Client-side JS).
* `utils/` – Error handling and helper utilities.
